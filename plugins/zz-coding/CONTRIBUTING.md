# 贡献指南

欢迎贡献新的设计模式！本指南帮助你了解如何添加新的 skill。

## 添加新 Skill 的步骤

### 1. 创建 Skill 目录

```
skills/
└── new-pattern-name/
    └── SKILL.md
```

### 2. 编写 SKILL.md

参考现有格式：

```yaml
---
name: new-pattern-name
description: 当用户讨论xxx话题时使用此skill。描述适用场景和提供模板。
version: 1.0.0
---

# 设计模式名称

## 适用场景
- 场景1
- 场景2

## 解决方案
描述解决方案

## 代码模板
提供可复用的代码模板，使用 {占位符} 表示需要替换的内容
```

### 3. 更新 README.md

在表格中添加新 skill 的说明。

### 4. 提交 PR

```
git add .
git commit -m "feat: 添加 xxx 设计模式"
git push
```

## Skill 描述规范

description 字段是关键，决定何时激活。建议包含：
- 用户可能说的关键词
- 具体场景描述
- 相关技术术语

示例：
```yaml
description: 当用户讨论活动TV动画展示、需要延迟展示数据、设计模式等话题时使用此skill。
```

## 版本管理

使用语义化版本号：
- 1.0.0 - 初始版本
- 1.1.0 - 新增内容
- 1.0.1 - Bug修复
- 2.0.0 - 重大变更

## 审核标准

- [ ] 描述清晰，能让其他开发者理解
- [ ] 包含实际可用的代码模板
- [ ] 说明适用场景和注意事项
- [ ] 不与现有 skill 重复
